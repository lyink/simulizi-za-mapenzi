## ğŸ¯ Key Design Patterns

### 1. Singleton Pattern
All services use singleton pattern for single instance:

```dart
class CacheService {
  static final CacheService _instance = CacheService._internal();
  factory CacheService() => _instance;
  CacheService._internal();
}
```

### 2. Cache-Aside Pattern
Data is loaded from cache first, then updated from source:

```dart
// 1. Try cache
final cached = await cacheService.getCachedStories();
if (cached != null) {
  return cached; // Fast path
}

// 2. Load from source
final stories = await firestore.collection('stories').get();

// 3. Update cache
await cacheService.cacheStories(stories);

return stories;
```

### 3. Observer Pattern
Real-time listeners for Firebase changes:

```dart
_firestore
  .collection('stories')
  .snapshots()
  .listen((snapshot) {
    // React to changes
  });
```

### 4. Strategy Pattern
Different notification strategies:

```dart
// Single story strategy
notifyNewStory(title, author, id)

// Multiple stories strategy
notifyMultipleNewStories(count)

// Reading reminder strategy
scheduleReadingReminder(hour, minute)
```
